# AGENTS.md

## 프로젝트 컨텍스트
아키텍처나 제품 관련 결정을 내리기 전에 `OVERVIEW.md`를 먼저 읽으세요.

## 계획 워크플로
- 현재 작업의 기준은 `KANBAN.md`입니다.
- 작업을 시작할 때: `진행 중` 항목을 먼저 확인하세요.
- 작업을 마칠 때: 해당 항목을 `KANBAN.md`의 변경 기록(CHANGELOG)으로 날짜와 함께 옮기세요.

## 검증
코드를 변경한 후, 완료를 보고하기 전에 관련된 검사를 실행하세요.

기본 검사:
- `<lint 명령어>`
- `<test 명령어>`
- `<build 명령어>`

비용이 큰 검사(필요할 때만):
- `<e2e 또는 전체 테스트 명령어>`

검사 명령어가 없거나, 차단되었거나, 너무 느리면 실행하지 말고 그 사실과 대신 실행한 내용을 명확히 보고하세요.

> 참고: 위 명령어는 `package.json`의 스크립트와 감지된 프레임워크를 기반으로 채워집니다. 기존 스크립트를 우선 사용하세요.
